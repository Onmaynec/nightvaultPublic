# NightVault v0.9.6

## Главное

Это техническое обновление перед следующим релизом: больше стабильности, понятнее диагностика подключения и надёжнее сборка через GitHub Actions.

## Что изменилось

- Добавлена команда `npm run check` для быстрой проверки JavaScript-файлов перед релизом.
- GitHub Actions переведён на Node.js 22 и больше не зависит от npm cache/lock-файла на этапе настройки Node.
- Перед сборкой теперь выполняются `npm run doctor` и `npm run check`.
- Добавлен endpoint `GET /api/health` для проверки сервера и подключения через браузер/Radmin VPN.
- Сервер теперь сохраняет `server/data.json` через временный файл и переименование, чтобы снизить риск повреждения базы.
- Улучшена обработка ошибок `fetch`: вместо молчаливого зависания показывается понятная ошибка подключения.
- WebSocket теперь пытается переподключиться автоматически после обрыва соединения.
- В настройках подключения добавлена кнопка «Проверить сервер».
- Обновлены README и инструкция релиза.
- Обновлено окно «Что нового» для версии 0.9.6.

## Как выпустить это обновление

```cmd
git add .
git commit -m "Release 0.9.6"
git push
git tag v0.9.6
git push origin v0.9.6
```

После успешной сборки GitHub Actions создай GitHub Release с тегом `v0.9.6` и прикрепи файлы из артефакта:

- `NightVault-Setup-0.9.6.exe`
- `latest.yml`
- `.blockmap` файлы, если есть

Без `latest.yml` автообновление не увидит новую версию.
